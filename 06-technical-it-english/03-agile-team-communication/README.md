# Agile, Standups & Team Communication (ការប្រាស្រ័យទាក់ទងក្នុងក្រុមការងារ Agile/Scrum)

នៅក្នុងបរិបទការងារអភិវឌ្ឍន៍កម្មវិធីបែបទំនើប ការប្រាស្រ័យទាក់ទងគ្នាក្នុងក្រុមការងារ (ជាពិសេសក្នុងកិច្ចប្រជុំប្រចាំថ្ងៃ Daily Standup) គឺចាំបាច់ណាស់ដើម្បីធានាថាការងារដើរបានលឿន និងគ្មានការស្ទះ។

In modern software development teams, clear communication—especially during Daily Standups—is vital to keep the pipeline moving and resolve blockers quickly.

---

## ⏰ ១. កិច្ចប្រជុំប្រចាំថ្ងៃ (Daily Standup Structure)

នៅក្នុង Daily Standup សមាជិកម្នាក់ៗត្រូវឆ្លើយសំណួរ ៣ យ៉ាងខ្លីៗ (ប្រើពេលមិនឱ្យលើសពី ២ នាទី)៖
1. **What did I do yesterday?** (តើខ្ញុំបានធ្វើអ្វីខ្លះកាលពីម្សិលមិញ?)
2. **What will I do today?** (តើខ្ញុំនឹងធ្វើអ្វីខ្លះនៅថ្ងៃនេះ?)
3. **Are there any blockers?** (តើមានបញ្ហាអ្វីរារាំង ឬធ្វើឱ្យការងារខ្ញុំយឺតយ៉ាវទេ?)

---

## 💬 ឃ្លាគន្លឹះសម្រាប់ប្រើប្រាស់ក្នុង Daily Standup (Key Standup Phrases)

### ក. បង្ហាញពីការងារកាលពីម្សិលមិញ (Yesterday's Progress)
* *"Yesterday, I completed the unit tests for the registration module."*
* *"I finished coding the frontend responsive layout."*
* *"Yesterday, I worked on database optimization and index creation."*

### ខ. បង្ហាញពីការងារថ្ងៃនេះ (Today's Plan)
* *"Today, I plan to focus on integrating the payment API."*
* *"I will start working on containerizing our services using Docker."*
* *"Today, I'll be reviewing Pull Requests from the frontend team."*

### គ. បង្ហាញពីបញ្ហារារាំង (Blockers)
* *"I have no blockers today."* (ថ្ងៃនេះខ្ញុំអត់មានបញ្ហាស្ទះអ្វីទេ។)
* *"I am blocked by a database credentials issue. I need assistance from the DevOps team."*
* *"Currently, I'm blocked because the third-party API is down. I'll follow up on this."*

---

## 🤝 ២. ការសុំជំនួយ និងការផ្តល់មតិយោបល់ (Asking for Help & Expressing Opinions)

### ការសុំជំនួយ (Asking for Help)
* *"Could you help me look at this error when you have a free moment?"*
* *"I'm stuck on this Hibernate mapping issue. Does anyone have experience with it?"*

### ការបញ្ចេញមតិយោបល់ដោយគួរសម (Expressing Opinions Politely)
* *"In my opinion, we should use PostgreSQL instead of MySQL because..."*
* *"I agree with your suggestion, but we might want to consider security performance too."*
* *"I see your point, however, I think that would delay our release schedule."*
