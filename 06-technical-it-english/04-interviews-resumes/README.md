# Tech Job Interviews & Resumes in English (ការរៀបចំ Resume និងការសម្ភាសន៍ការងារ IT)

ការរៀបចំប្រវត្តិរូបសង្ខេប (Resume) ជាភាសាអង់គ្លេសប្រកបដោយភាពទាក់ទាញ និងការត្រៀមខ្លួនសម្រាប់សម្ភាសន៍ការងារ គឺជាគន្លឹះដើម្បីទទួលបានឱកាសការងារល្អៗ ទាំងក្នុងស្រុក និងអន្តរជាតិ។

Preparing a compelling English IT resume and practicing interview responses are crucial steps to securing high-paying software engineering positions globally.

---

## 📄 ១. ការរៀបចំ Resume ផ្នែក IT (IT Resume Structure)

Resume បច្ចេកវិទ្យាគួរតែមានភាពច្បាស់លាស់ ងាយស្រួលអាន (និយមប្រើប្រាស់ទម្រង់ ១ ទំព័រ) និងមានផ្នែកសំខាន់ៗដូចខាងក្រោម៖
1. **Contact Information:** ឈ្មោះ លេខទូរសព្ទ អ៊ីមែល គណនី GitHub និង LinkedIn។
2. **Professional Summary:** សង្ខេបពីបទពិសោធន៍ និងជំនាញចម្បងរបស់អ្នក (២-៣ ជួរ)។
3. **Technical Skills:** ចែកជាផ្នែកៗ (Languages, Frameworks, Databases, Tools)។
4. **Work Experience:** ចាប់ផ្តើមពីការងារបច្ចុប្បន្នថយក្រោយ។ ប្រើ **Action Verbs** (កិរិយាសព្ទសកម្មភាព) ក្នុងអតីតកាល។
5. **Projects:** គម្រោងផ្ទាល់ខ្លួន ឬគម្រោងសំខាន់ៗដែលបង្ហាញពីសមត្ថភាពពិត។
6. **Education:** សញ្ញាបត្រ ជំនាញ គ្រឹះស្ថានសិក្សា និងឆ្នាំបញ្ចប់។

## ⚡ កិរិយាសព្ទសកម្មភាពសម្រាប់ Resume (Action Verbs for Work Experience)

កុំសរសេរថា *"I was responsible for coding..."*។ ផ្ទុយទៅវិញ ត្រូវប្រើប្រាស់ Action Verbs ខ្លាំងៗ៖
* **Designed:** Designed a microservices-based API gateway. (បានរចនា API Gateway ផ្អែកលើ microservices)
* **Developed:** Developed responsive web applications using React. (បានបង្កើតកម្មវិធីគេហទំព័រដែលឆ្លើយតបនឹងឧបករណ៍ប្រើប្រាស់ React)
* **Implemented:** Implemented secure JWT user authentication logic. (បានអនុវត្តប្រព័ន្ធផ្ទៀងផ្ទាត់គណនីអ្នកប្រើប្រាស់ JWT)
* **Optimized:** Optimized SQL queries, reducing page load time by 30%. (បានបង្កើនប្រសិទ្ធភាពសំនួរទិន្នន័យ SQL កាត់បន្ថយពេល load ទំព័រ ៣០%)
* **Collaborated:** Collaborated with cross-functional teams using Agile. (បានសហការជាមួយក្រុមការងារចំរុះមុខងារដោយប្រើប្រាស់ Agile)

## 🗣️ ២. គន្លឹះឆ្លើយសំណួរសម្ភាសន៍ការងារ (Acing the Job Interview)

នៅពេលឆ្លើយសំណួរអំពីបទពិសោធន៍ ឬបញ្ហាដែលអ្នកធ្លាប់ជួបប្រទះ គួរប្រើប្រាស់ **STAR Method** (វិធីសាស្ត្រផ្អែកលើករណីជាក់ស្តែង)៖

* **S - Situation (ស្ថានភាព):** ពណ៌នាពីស្ថានភាព ឬបរិបទដែលអ្នកជួបប្រទះ។
* **T - Task (ភារកិច្ច):** ភារកិច្ច ឬគោលដៅដែលត្រូវសម្រេច។
* **A - Action (សកម្មភាព):** សកម្មភាពដែលអ្នកបានធ្វើផ្ទាល់ខ្លួនដើម្បីដោះស្រាយបញ្ហានោះ។
* **R - Result (លទ្ធផល):** លទ្ធផលដែលទទួលបាន (បើមានទិន្នន័យជាលេខ ឬភាគរយកាន់តែល្អ)។

### សំណួរទូទៅ៖ *"Tell me about a difficult bug you fixed."*
> **(Situation)** "On my last project, the production server crashed during a peak traffic hour due to database connection exhaustion.
> **(Task)** My task was to identify the memory leak and ensure the database pool was properly managed.
> **(Action)** I analyzed the heap dump using Chrome DevTools/JVM Profiler and found that DB connections were not being closed inside the resource block. I immediately rewritten the helper to use a try-with-resources statement to guarantee execution closure.
> **(Result)** As a result, database resource usage decreased by 40%, and the system maintained 100% uptime during subsequent peak traffic events."
